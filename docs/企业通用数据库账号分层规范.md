# 企业通用数据库账号分层规范（兼容 MySQL / Oracle / PostgreSQL / TiDB）

## 文档说明

1. 统一分为**9 大类账号**，覆盖 OLTP 关系型数据库全场景，一套标准适配 MySQL、Oracle、PostgreSQL、TiDB；
2. 按权限从高到低分层管控，遵循最小权限、权限隔离、操作审计原则；
3. 每种账号包含：定位、通用管控规则、各数据库差异化权限配置；
4. 适用于权限工单、账号基线巡检、安全审计、数据库标准化落地。

## 目录

1. 超级管理员
2. 高级 DBA 用户
3. 日常运维用户
4. 业务应用账号
5. 只读分析账号

9. 系统备份账号

## 1. 超级管理员

### 核心定位

数据库最高权限账号，仅用于初始化部署、重大故障急救、介质恢复、底层内核参数调整；**禁止日常业务、日常运维登录使用**。

### 通用管控规则

- 仅允许堡垒机 / 机房本地 IP 白名单访问，关闭公网远程登录；
- 高危操作执行双人复核，全程开启审计日志；
- 禁止程序、报表、开发人员持有该账号。

### 各数据库账号与权限

表格

|   数据库   |      内置账号       |                         核心权限说明                         |
| :--------: | :-----------------: | :----------------------------------------------------------: |
|   MySQL    |        root         | 全局 ALL PRIVILEGES，可启停实例、修改文件、创建删除所有对象  |
|   Oracle   | SYS(SYSDBA)、SYSTEM | SYS 拥有内核级权限，可启停库、介质恢复、修改控制文件；SYSTEM 为次一级系统管理员 |
| PostgreSQL |      postgres       | 自带 SUPERUSER 属性，可创建角色、表空间、修改全局参数、恢复数据库 |
|    TiDB    |        root         | 集群最高权限，可管控 PD 元数据、调整集群调度、全局 DDL 开关、全库权限分配 |

------

## 2. 高级 DBA 用户

### 核心定位

资深 DBA / 数据库架构师使用，负责数据库规划、容量扩容、表空间 / Schema 管理、账号权限分配、重大结构变更。

### 通用管控规则

- 不授予超级管理员内核权限（SYSDBA/SUPERUSER/ 集群底层管控权限）；
- 删库、清空大表、分区删除等高危操作需工单审批；
- 可批量创建业务账号、分配基础权限。

### 各数据库账号与权限

表格

|   数据库   |      自定义账号       |                         核心权限说明                         |
| :--------: | :-------------------: | :----------------------------------------------------------: |
|   MySQL    |       dba_admin       | 全局大部分权限，剔除 SHUTDOWN、FILE 高危权限；可建库、建用户、执行 DDL |
|   Oracle   | DBA_ADMIN / CDB_ADMIN | 授予 DBA 角色，无 SYSDBA/SYSOPER；支持表空间、用户、存储过程、AWR 性能分析 |
| PostgreSQL |       db_admin        | 授予 CREATEDB、CREATEROLE，按需开启 SUPERUSER；管理 Schema、索引、分区、表空间 |
|    TiDB    |       tidb_dba        | 全库普通权限，可创建 Database、分配账号、调整会话参数，无集群底层修改权限 |

------

## 3. 日常运维用户

### 核心定位

普通运维、开发 DBA 线上排查使用，慢 SQL 分析、会话查杀、索引优化、临时测试表创建，无数据删除 / 删库高危权限。

### 通用管控规则

- 无 DROP DATABASE、DROP TABLE、TRUNCATE 全局权限；
- 仅支持查询性能视图、终止异常会话、创建临时表；
- 不允许批量清理业务数据。

### 各数据库账号与权限

表格

|   数据库   | 自定义账号 |                         核心权限说明                         |
| :--------: | :--------: | :----------------------------------------------------------: |
|   MySQL    |  ops_ops   | PROCESS、SELECT、ALTER、CREATE TEMPORARY、SHOW VIEW；禁止 FILE、DROP 全局权限 |
|   Oracle   |  OPS_DBA   | SELECT_CATALOG_ROLE、会话管理、基础表 DDL；无删除表空间、删除用户权限 |
| PostgreSQL |   ops_pg   | 查询 pg_catalog、pg_stat_* 视图，pg_terminate_backend；仅自有 Schema 基础 DDL |
|    TiDB    |  ops_tidb  | PROCESS、查询 INFORMATION_SCHEMA、kill 会话、创建临时表，无删库权限 |

------

## 4. 业务应用账号

### 核心定位

业务程序服务连接数据库，分为**长期读写账号**和**上线临时 DDL 账号**，严格最小权限隔离。

### 通用管控规则

1. 一个业务模块一套独立账号，跨业务数据隔离；
2. 日常读写账号仅 DML 权限，无任何 DDL；
3. DDL 账号仅上线窗口启用，发布完成后立即回收 CREATE/ALTER/DROP 权限；
4. 禁止授予全局 ANY/ALL 权限，仅授权对应业务库 / Schema。

### 各数据库账号与权限

表格

|   数据库   |               账号划分               |                         核心权限说明                         |
| :--------: | :----------------------------------: | :----------------------------------------------------------: |
|   MySQL    | app_xxx（读写）、app_ddl_xxx（发布） | app_xxx：SELECT/INSERT/UPDATE/DELETE；app_ddl_xxx 临时授予 CREATE/ALTER |
|   Oracle   |         app_xxx、app_ddl_xxx         | 仅本 Schema DML；DDL 账号临时授予表、索引、存储过程创建权限  |
| PostgreSQL |         app_xxx、app_ddl_xxx         |  目标 Schema USAGE + DML；发布期授予 CREATE/ALTER，到期回收  |
|    TiDB    |         app_xxx、app_ddl_xxx         |      权限语法同 MySQL，按 Database 隔离权限，不跨库访问      |

------

## 5. 只读分析账号

### 核心定位

报表平台、BI、数据分析、业务查询使用，**无任何写操作权限**，仅查询数据。

### 通用管控规则

- 禁止 INSERT/UPDATE/DELETE/TRUNCATE/LOCK 写入类权限；
- 大批量敏感数据导出需单独使用【数据导出账号】走审批；
- 限制大查询并发，避免拖垮业务库。

### 各数据库账号与权限

表格

|   数据库   |  自定义账号   |                    核心权限说明                     |
| :--------: | :-----------: | :-------------------------------------------------: |
|   MySQL    |   read_xxx    |          仅 SELECT，无锁表、修改、删除权限          |
|   Oracle   | READ_ONLY_XXX | SELECT_CATALOG_ROLE + 业务表查询权限，无 DML 写权限 |
| PostgreSQL |   read_xxx    |   Schema USAGE + SELECT，禁止 COPY 写入、DML 修改   |
|    TiDB    |   read_xxx    |         仅 SELECT，无 LOCK TABLES、DML 权限         |

------

## 6. 数据复制账号

### 核心定位

用于主从同步、灾备同步、跨库数据同步、实时数据订阅组件。

### 通用管控规则

- 仅分配同步日志 / 归档所需最小权限，不开放业务数据修改权限；
- 主备库 / 上下游集群统一账号，IP 白名单限制同步节点。

### 各数据库账号与权限

表格

|   数据库   |   自定义账号    |                         核心权限说明                         |
| :--------: | :-------------: | :----------------------------------------------------------: |
|   MySQL    |      repl       |      REPLICATION SLAVE、REPLICATION CLIENT，读取 binlog      |
|   Oracle   | DG_REP、OGG_REP | DG_REP 用于 Data Guard 主备；OGG_REP 适配 GoldenGate 数据同步 |
| PostgreSQL |     replica     |          REPLICATION 专属权限，用于 WAL 流复制主备           |
|    TiDB    |      repl       |     REPLICATION 权限，供 TiCDC 实时同步、上下游数据订阅      |

------

## 7. 数据导出账号

### 核心定位

离线取数、业务逻辑导出、临时数据提取，仅支持查询导出，无数据修改能力。

### 通用管控规则

- 仅读取权限，无写入、删除、更新权限；
- 敏感业务数据导出需流程审批，导出日志留存；
- 区分单业务导出、全库导出两类权限。

### 各数据库账号与权限

表格

|   数据库   | 自定义账号 |                       核心权限说明                        |
| :--------: | :--------: | :-------------------------------------------------------: |
|   MySQL    |  exp_user  |       SELECT、LOCK TABLES，适配 mysqldump 逻辑导出        |
|   Oracle   |  EXP_USER  | EXP_FULL_DATABASE / 单 Schema 查询权限，支持 expdp 数据泵 |
| PostgreSQL |  exp_user  |           SELECT、COPY 读权限，禁止 COPY 写入表           |
|    TiDB    |  exp_user  |        SELECT、LOCK TABLES，适配 Dumpling 导出工具        |

------

## 8. 系统监控账号

### 核心定位

监控平台（Prometheus/Zabbix/Grafana/EM）采集性能指标、会话、锁、IO、慢 SQL 数据。

### 通用管控规则

- 纯只读账号，无任何修改、DDL、DML 权限；
- 仅开放性能视图、系统字典查询权限；
- 禁止 kill 会话、修改全局参数。

### 各数据库账号与权限

表格

|   数据库   |  自定义账号  |                       核心权限说明                       |
| :--------: | :----------: | :------------------------------------------------------: |
|   MySQL    |   monitor    |           PROCESS、SELECT 性能视图，无修改权限           |
|   Oracle   | MONITOR_USER |    SELECT_CATALOG_ROLE，仅查询 v$ 性能视图、AWR 快照     |
| PostgreSQL |   monitor    |           查询 pg_stat_*、pg_catalog 系统视图            |
|    TiDB    |   monitor    | 查询 INFORMATION_SCHEMA、performance_schema 性能指标视图 |

------

## 9. 系统备份账号

### 核心定位

物理备份、归档备份、全库离线备份，适配各数据库原生备份工具。

### 通用管控规则

- 权限隔离：仅备份所需权限，不开放业务数据修改；
- 物理备份与逻辑导出账号分离，不混用；
- 备份操作日志留存，定期校验备份文件可用性。

### 各数据库账号与权限

表格

|   数据库   | 自定义账号 |                        核心权限说明                         |
| :--------: | :--------: | :---------------------------------------------------------: |
|   MySQL    |  bak_user  |       LOCK TABLES、RELOAD，适配 XtraBackup、mysqldump       |
|   Oracle   |  BAK_USER  | 授予 SYSOPER 角色，用于 RMAN 物理备份；无业务表明细访问权限 |
| PostgreSQL |  bak_user  |       BACKUP 专属权限，支持 pg_basebackup 流物理备份        |
|    TiDB    |  bak_user  |          集群元数据读取权限，适配 BR 集群备份工具           |

# 通用落地规范（全数据库统一）

## 1. 账号命名规范

- 超级管理员：原生内置账号，禁止日常使用
- 管理类：`dba_admin`、`tidb_dba`、`db_admin`、`ops_ops`
- 业务类：`app_业务模块`、`app_ddl_业务模块`、`read_业务模块`
- 工具类：`repl`、`exp_user`、`monitor`、`bak_user`

## 2. 权限治理统一原则

1. **最小权限**：业务账号仅访问自身业务对象，不授予全局 ANY/ALL 权限；
2. **权限时效分离**：DDL 变更单独账号，仅上线窗口开放，其余时间回收；
3. **账号隔离**：备份、复制、监控、导出账号独立拆分，不共用；
4. **IP 白名单**：所有账号绑定访问来源 IP，超级管理员严格限制内网堡垒机；
5. **全量审计**：开启登录审计、DROP/TRUNCATE/DELETE 大批量数据高危操作审计。

## 3. 数据库差异化兼容说明

1. 角色体系

   

   Oracle、PostgreSQL 原生支持 Role 角色；MySQL 8.0+/TiDB 支持角色，低版本 MySQL 直接通过账号权限分组模拟角色管理；

2. 复制架构

   

   MySQL/TiDB 基于 binlog；PG 基于 WAL 流复制；Oracle 分 DG 物理备库、OGG 逻辑同步，统一归类为「数据复制账号」；

3. 备份工具差异

   

   MySQL：XtraBackup；Oracle：RMAN；PG：pg_basebackup；TiDB：BR；统一归类「系统备份账号」；

4. 多租户特性

   

   Oracle 12c CDB/PDB 区分公共 / 本地用户；MySQL/PG/TiDB 无容器租户，仅增加配套管控规则，不改动 9 大类分层结构。

## 4. 适用范围与不适用范围

### 适用

MySQL、Oracle、PostgreSQL、TiDB、OceanBase、PolarDB、达梦等主流关系型 OLTP 数据库

### 不适用

Redis、MongoDB、InfluxDB 等键值、文档、时序非关系型数据库
