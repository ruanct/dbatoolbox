## 1. 基础版本定位与生命周期

### 相同点

- 均支持 RAC 集群、ASM 自动存储管理、Data Guard 物理备库、分区表、透明数据加密 TDE
- 均支持 AMM 自动内存管理、AWR/ADDM 性能诊断、RMAN 全量 / 增量备份

### 不同点

表格







|     对比项     |              Oracle 11gR2 (11.2.0.4)               |            Oracle 19c (12.2.0.3 LTS 长期支持版)            |
| :------------: | :------------------------------------------------: | :--------------------------------------------------------: |
|    发布定位    |        网格计算时代，单实例 No-CDB 架构为主        |          云原生 LTS 稳定版，CDB 多租户为核心架构           |
|  官方支持周期  | 标准支持 2015 结束，扩展支持 2020 终止，无官方补丁 |     标准支持至 2027，付费扩展支持至 2030，持续安全补丁     |
|   数据库架构   |        仅 No-CDB 独立单库，无容器 PDB 概念         | 主推 CDB 多租户，兼容传统 No-CDB 模式（21c 起移除 No-CDB） |
| 客户端兼容策略 |         兼容 9i/10g/11g 低版本 OCI 客户端          | 默认拒绝 11g 及以下老旧客户端，需修改`sqlnet.ora`关闭校验  |

## 2. 底层核心架构（代际核心差异）

### 相同点

- 后台进程模型（PMON/SMON/DBWR/LGWR/CKPT）、redo/undo 事务机制、MVCC 读一致性底层逻辑不变
- 数据文件、控制文件、在线重做日志基础存储模型完全通用

### 不同点

1. CDB/PDB 多租户架构（11g 完全不具备）
   - 11g：一个数据库对应一套独立实例，多业务库需部署多套数据库，资源无法共享、隔离薄弱；全局共享 undo 表空间
   - 19c：一套 CDB 容器承载多个可插拔 PDB，共享 ASM、集群、后台进程；支持 PDB 克隆、热克隆、拔出 / 插入、快照轮播、PDB 独立闪回；每个 PDB 拥有私有 undo 表空间
2. 分层数据字典
   - 11g：全局统一数据字典，无租户隔离
   - 19c：根 CDB 共享字典 + 各 PDB 私有本地字典，PDB 之间元数据完全隔离
3. 自治数据库底座能力
   - 11g：无自治运维能力，调优、补丁、备份、清理全部人工操作
   - 19c 内置自治能力：自动索引、自动统计信息采集、自动闪回日志清理、GI 零停机滚动补丁

------

## 3. 内存管理体系

### 相同点

- 均支持`MEMORY_TARGET` AMM 全自动内存、`SGA_TARGET` ASMM 自动 SGA、`PGA_AGGREGATE_TARGET`
- Buffer Cache、Shared Pool、Large Pool、Java Pool 内存组件概念一致

### 不同点

1. PDB 内存资源隔离（19c 独有）

   

   19c 可通过资源管理器为每个 PDB 配置独立 SGA/PGA 上限，实现业务租户内存隔离；11g 无租户概念，内存全局共享无管控

2. PGA 内存硬性保护

   - 11g：`pga_aggregate_limit`为软限制，易出现 PGA 溢出导致实例 OOM
   - 19c 强化`pga_aggregate_limit`硬性管控，超限会话自动终止，保护实例稳定

3. In-Memory 内存列存储选件

   - 11g：完全不支持 IM 列存储
   - 19c 完整支持 IM，备库 ADG 同步时可同时启用 IM，支撑 HTAP 混合负载

4. SGA 在线动态调优优化

   

   19c 调整 SGA 大小无业务抖动，内存碎片回收机制优化

------

## 4. SQL、数据类型、DDL 执行能力

### 相同点

- 基础 DML、标准 DDL、事务隔离级别、关联查询、内置函数基础语法兼容
- 分区表、B 树索引、约束、存储过程、触发器底层执行逻辑通用

### 不同点

1. 快速添加带 DEFAULT 字段（运维重大差异）

   - 11g：大表新增带 DEFAULT 字段会全表重写数据，亿级表耗时极长、长期锁表；仅`DEFAULT NOT NULL`走元数据优化
   - 19c：所有带 DEFAULT 字段新增仅修改元数据，秒级执行；分区表、压缩表全部支持快速 DDL，无表空间膨胀

2. 超长字符串类型

   - 11g：VARCHAR2/NVARCHAR2 最大 4000 字节
   - 19c 开启`MAX_SQL_STRING_SIZE=EXTENDED`后，VARCHAR2 最大支持 32767 字节

3. 自动索引 Automatic Indexing（19c 独有）

   

   根据业务负载自动创建、维护、清理无用索引，减少人工 SQL 调优成本，11g 无此功能

4. 原生 JSON 完整支持

   - 11g：仅少量简易 JSON 函数，无 JSON 专用索引、JSON_TABLE、并行扫描
   - 19c 原生 JSON 数据类型、JSON 索引、JSON_TABLE、聚合函数、并行查询优化，兼容 NoSQL 场景

5. 隐藏列 Invisible Column

   - 11g：无隐藏字段特性
   - 19c 支持建表定义隐藏列，适配业务灰度升级、平滑改造

6. 递归 CTE、标准窗口函数

   - 11g 不支持递归 WITH 与完整窗口函数；19c 完全兼容标准 SQL 窗口函数、递归表达式

------

## 5. 分区、In-Memory 内存列存储

### 相同点

11g 支持范围、列表、哈希、复合分区，支持本地 / 全局分区索引

### 不同点

1. 分区功能全面增强
   - 19c 支持自动间隔分区、虚拟列分区、区间 - 列表复合分区、分区在线迁移、分区自动归档
   - 19c 分区 DDL 在线执行无独占锁；11g 大分区操作会长时间锁表阻塞业务
2. In-Memory 列存储 IM
   - 11g：无该选件
   - 19c 支持多种 IM 压缩级别、后台快速填充 IM、PDB 独立 IM 内存配额，兼顾 OLTP 联机业务与 OLAP 统计查询
3. 分区级闪回、分区独立备份恢复：19c 原生支持，11g 仅能整库闪回

------

## 6. 高可用 Data Guard / RAC / ASM

### 相同点

- 均支持物理备库、逻辑备库、RAC 多节点集群、ASM 磁盘组、故障组、GI 滚动补丁
- 同步 / 异步 redo 传输、快照备库、闪回数据库基础能力通用

### 不同点

#### 6.1 Active Data Guard 升级

1. 备库 DML 重定向（19c 独有）

   

   备库执行写入 DML 会自动转发至主库，实现透明读写分离；11g 备库仅只读，写入必须切换至主库

2. 还原点跨主备同步，主库闪回后备库自动联动闪回，11g 闪回后主备位点断裂需重建备库

3. 多进程并行 redo apply，备库日志应用速度大幅提升，显著降低主从延迟

#### 6.2 RAC & Grid Infrastructure

1. 19c 支持**零停机 GI 滚动补丁**，集群打补丁全程无业务中断；11g GI 补丁需要节点停机维护
2. DRM 动态资源重平衡逻辑优化，11g 频繁 DRM 易引发性能抖动、节点驱逐宕机
3. 新增 PDB 粒度 RAC 资源管控，11g 无租户隔离能力
4. 心跳、网络超时容错策略优化，节点驱逐逻辑更稳定

#### 6.3 ASM 存储

19c 支持磁盘组弹性扩容、平衡 IO 限流、ASM 快照轮播、元数据自动备份；11g 磁盘重平衡会抢占业务 IO 资源

------

## 7. 备份恢复 RMAN、闪回技术

### 相同点

- RMAN 全量 / 增量 / 归档备份、压缩备份、跨平台传输表空间、闪回查询、闪回表、闪回数据库基础功能通用

### 不同点

1. RMAN 多租户适配
   - 11g：只能整库备份，无法单独备份单个业务库
   - 19c 支持 PDB 单独备份、PDB 级恢复目录、虚拟私有恢复目录，租户备份权限精细化隔离
2. 闪回日志自动生命周期管理
   - 11g：闪回日志持续膨胀，容易占满存储，需要人工定期清理
   - 19c 根据备份保留策略自动清理过期闪回日志，限制存储占用
3. 可传输备份 Transportable Backup：19c 支持跨 CDB/PDB 快速迁移，11g 仅支持整库迁移
4. ADG 备库增量同步优化：19c 增量同步无需重建备库，11g 流程繁琐耗时

------

## 8. 安全、审计、账号权限体系

### 相同点

- TDE 透明数据加密、表空间加密、细粒度审计、角色权限、密码复杂度基础策略一致

### 不同点

1. Schema Only 纯模式账号（19c 独有）

   

   账号无登录权限，仅用于存储过程、ETL 后台任务，缩小攻击面，11g 无此类账号

2. 特权分析 Privilege Analysis

   

   自动采集账号实际使用权限，清理冗余高危权限，满足等保合规，11g 无该功能

3. 密码安全策略强化

   

   19c 支持历史密码黑名单、密码复用限制、精细化有效期管控；11g 仅简单过期策略

4. 审计体系隔离

   - 11g 审计日志统一存储在 SYSTEM 表空间，极易膨胀
   - 19c 审计数据独立表空间、PDB 独立审计、自动归档清理审计日志

5. 网络传输加密

   

   19c 默认启用 TLSv1.2，废弃弱加密套件；11g 默认明文传输，SSL 配置复杂

------

## 9. 运维自动化、诊断监控

### 相同点

AWR、ADDM、ASH 性能视图、EM 管理工具、SQL Tuning Advisor、RAT 真实应用测试框架通用

### 不同点

1. 自治自动化运维

   - 11g：统计信息、索引维护、闪回清理、补丁全部人工操作
   - 19c 自动统计信息收集、自动索引、自动闪回日志清理、内存自动调优

2. 租户独立监控能力

   

   19c 新增 PDB 独立 AWR、PDB 粒度 ASH、多租户专属性能视图；11g 仅全局实例监控，无法隔离单业务负载

3. 诊断能力增强

   

   19c ADDM 支持租户负载分析、自动捕获阻塞事务、死锁自动归档、OOM 内存诊断报告；11g 诊断粒度粗

4. Easy Connect 简化连接串，无需本地配置 tnsnames.ora

------

## 10. JSON、分片、云原生扩展能力

### 相同点

均支持 Exadata 一体机部署、数据泵 expdp/impdp、外部表访问

### 不同点

1. Oracle Sharding 分布式分片（11g 完全无）

   

   19c 支持分布式分片数据库，海量数据水平拆分，支持多表分片家族、分片在线扩容，适配高并发交易、IoT 业务

2. 云原生 DBaaS 适配

   - 11g 无云原生租户架构，无法弹性资源隔离
   - 19c 适配 Oracle 公有云自治 ATP/ADW、混合云迁移、跨云 Data Guard 异地灾备

3. 大数据原生连接器：19c 可直接关联查询 HDFS/Hive 数据，11g 无原生大数据对接能力

------

## 11. 两者完全一致、无改动核心功能

1. ACID 事务、MVCC 多版本读一致性、行级锁、意向锁底层实现逻辑
2. 基础 DML、单表标准 DDL、主键 / 唯一 / 联合 BTree 索引机制
3. ASM 磁盘组、RAC 多节点集群基础通信、Data Guard 物理备库核心 redo 传输逻辑
4. RMAN 增量备份、数据泵导入导出、闪回查询 / 闪回表基础能力
5. AWR/ASH/ADDM 性能诊断框架、SQL 调优顾问
6. TDE 透明表空间加密、基础角色与权限控制模型
7. 分区表基础类型（范围 / 哈希 / 列表）、约束、存储过程、触发器底层执行
8. 重做日志 redo、撤销段 undo 事务崩溃恢复底层机制

------

## 12. 11g 存在、19c 废弃 / 不兼容特性

1. 客户端兼容限制：11g/12.1 老旧 OCI 客户端默认无法连接 19c，需修改 sqlnet 关闭安全校验
2. 废弃 Database Control 本地企业管理器，19c 仅保留轻量化 EM Express
3. 废弃旧版 XMLType 存储格式，19c 统一二进制存储 JSON/XML 数据
4. 大量 11g 专属隐含 DRM 调参在 19c 失效，参数逻辑重构
5. 11g No-CDB 单库无法直接原地升级至 19c CDB，需转换为 PDB
6. 废弃部分老旧审计视图、过时外部表访问驱动
7. Oracle Streams 流复制逐步废弃，19c 主推 GoldenGate 复制工具

------

## 13. 升级迁移关键风险点

1. 多租户架构改造：11g 无容器概念，升级后需将原库转换为 PDB，迁移流程复杂度提升
2. 客户端驱动兼容：应用、ETL、备份脚本使用 11g OCI 客户端会报 ORA-28040 认证不匹配
3. DDL 行为变更：11g 耗时的大表加字段 DDL 在 19c 秒级完成，若业务依赖锁表阻塞逻辑需要改造
4. 密码策略收紧：19c 密码复杂度、复用、过期规则更严格，老旧弱密码账号需要批量整改
5. ADG 备库 DML 重定向：原有备库只读业务写入逻辑需要适配新转发机制
6. 内存参数隔离：19c PDB 内存配额参数与 11g 全局内存配置逻辑不通用，需要重新压测调优
7. JSON 语法兼容：11g 简易 JSON 函数部分语法在 19c 重构，少量 SQL 存在兼容问题
