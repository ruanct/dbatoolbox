## 十一、兼容性（11g 代码迁移注意事项）

### 相同兼容逻辑

1. DML 基础增删改查、标准 PL/SQL 块、基础函数、事务、锁机制 100% 兼容；
2. ASM、RMAN、Data Guard 基础配置语法向下兼容；
3. 绝大多数 11g 存储过程、函数、触发器无需修改即可运行；
4. OS 认证、TNS 监听、ODBC/JDBC 驱动连接逻辑通用。

### 19c 不兼容 11g 的场景（必须改造）

1. 使用`wm_concat`函数 → 替换 LISTAGG；
2. 依赖 Streams 同步 → 迁移 GoldenGate；
3. 使用 Multimedia 图片处理组件 → 重构存储逻辑；
4. 弱密码 / 加密协议（DES/MD5）→ 升级客户端驱动；
5. 依赖 30 字符超长对象名 → 19c 需缩短或开启 128 字符模式；
6. 大量使用 DBMS_JOB → 迁移至 DBMS_SCHEDULER；
7. 依赖传统审计视图 DBA_AUDIT_SESSION（开启统一审计后无数据）→ 切换 UNIFIED_AUDIT_TRAIL。

## 十二、性能优化核心异同

### 相同优化机制

1. 基于成本优化器 CBO、索引、分区裁剪、并行查询；
2. Buffer Cache、Shared Pool、PGA 内存自动调优；
3. AWR/ASH/ADDM 性能诊断体系；
4. 表压缩、索引压缩、SecureFile LOB 优化。

### 19c 独有性能增强（11g 无）

1. VARCHAR2 32767 大字段减少 LOB 使用；
2. 大表新增 DEFAULT 字段秒级元数据优化；
3. In-Memory 列式存储混合负载加速 OLAP；
4. Query Quarantine SQL 隔离，防止慢 SQL 拖垮数据库；
5. ADG 多进程并行重做应用，同步速度提升数倍；
6. RAC 故障中断时间缩短 75%；
7. 自动索引、实时统计自动维护（Exadata）；
8. 混合分区冷热分层，归档查询性能提升。

## 十三、总结总览

### 核心相同点

1. 底层事务、存储、重做、ASM、RAC、Data Guard 基础架构一脉相承；
2. 标准 SQL、PL/SQL、RMAN、数据泵、权限角色、闪回核心功能完全兼容；
3. 运维基础命令、监听、参数文件、备份恢复流程逻辑统一；
4. 企业级高可用、容灾、分区、压缩能力全部继承。

### 核心差异总览

1. **架构代差**：11g 单实例 Non-CDB；19c 原生 CDB 多租户，支持云化多库隔离；
2. **数据能力**：19c 支持 32767 VARCHAR2、原生 JSON、混合分区、秒级 DDL；
3. **高可用升级**：ADG 备库读写、并行重做、零停机 GI 补丁、RAC 极速故障恢复；
4. **安全审计**：统一审计、权限分析、强加密强制、Schema 仅账号；
5. **自动化自治**：自动索引、SQL 隔离、PDB 级性能诊断、升级自动化；
6. **淘汰老旧组件**：Streams、Multimedia、弱加密、wm_concat 全部移除；
7. **生命周期**：11g 官方支持已结束，19c 长期支持至 2027，适合长期稳定生产。
